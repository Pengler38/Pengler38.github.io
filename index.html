About the Hello World banner

This is a 3D banner I programmed. I wanted to write a flashy and complex project in Rust that worked with graphics libraries at as low of a level as is reasonable.

And I'm satisfied with how it turned out! Instead of a higher level library like a game engine or three.js, I used wgpu, the Rust implementation of the WebGPU API. It's certainly not the easiest way to get a good looking result, but I learned the most doing it.

Points of interest:

  • Can be built for desktop or WebAssembly to be used on a website (Such as this one).
  • You can mouse over the letters and they will deform slightly, showing that it's a running program and not an animation.
  • The letter models and letter texture aren't loaded assets, they're generated as the program starts up. The letters are made with handwritten points and a few functions to mirror and duplicate them, while the texture is generated by a simple noise function.
  • In the build process I have a build.rs script which counts the number of lines in *.rs files and adds that into the program as a constant, which it can display on the website.
  • Power consumption should be low even on mobile devices. The CPU is hardly doing any work after startup and all the runtime complexity is in the vertex and fragment shaders.
